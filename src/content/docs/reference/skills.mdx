## Intent commands (11)

The primary routing commands — describe an intent, the skill picks the right document type or multi-step flow.

| Command                 | Purpose                                                                         |
| ----------------------- | ------------------------------------------------------------------------------- |
| `/archcore:context`     | Load the rules, ADRs, specs, and patterns that apply to a code area before changing it. |
| `/archcore:bootstrap`   | First-time onboarding — scale-aware seeding for an empty `.archcore/`.         |
| `/archcore:capture`     | Document a module, component, or API — routes to adr, spec, doc, or guide.     |
| `/archcore:plan`        | Plan a feature — routes to `product-track` or a single plan.                   |
| `/archcore:decide`      | Record a finalized decision — creates ADR, offers rule + guide follow-up.      |
| `/archcore:standard`    | Establish a team standard — drives `standard-track`.                           |
| `/archcore:review`      | Documentation health audit — gaps, staleness, orphans, missing relations.      |
| `/archcore:status`      | Counts dashboard — by category, type, status, relation coverage.               |
| `/archcore:actualize`   | Detect stale docs via code drift, cascade analysis, temporal signals.          |
| `/archcore:graph`       | Render the document relation graph as a Mermaid flowchart.                     |
| `/archcore:help`        | List every skill grouped by family.                                            |

## Tracks (6)

Multi-step workflows — each orchestrates several documents and wires relations automatically.

| Command                         | Flow                                       | Use when                                                     |
| ------------------------------- | ------------------------------------------ | ------------------------------------------------------------ |
| `/archcore:product-track`       | idea → prd → plan                          | Lightweight product flow — simple and fast.                  |
| `/archcore:sources-track`       | mrd → brd → urd                            | Discovery-focused — market, business, user inputs.           |
| `/archcore:iso-track`           | brs → strs → syrs → srs                    | Formal ISO 29148 cascade with traceability.                  |
| `/archcore:architecture-track`  | adr → spec → plan                          | Design decisions flowing into implementation.                |
| `/archcore:standard-track`      | adr → (optional cpat) → rule → guide       | Decision → codified standard → how-to.                       |
| `/archcore:feature-track`       | prd → spec → plan → task-type              | Full feature lifecycle.                                      |

## Utility

| Command               | Purpose                                                                        |
| --------------------- | ------------------------------------------------------------------------------ |
| `/archcore:verify`    | Plugin integrity check — validates configs, scripts, skills, hooks, and runs the test suite. |

## Document types

The 18 document types no longer have dedicated slash commands. Use intent commands and tracks above, or call `mcp__archcore__create_document` directly when you need a specific type without a routing skill.

| Layer      | Types                                                              |
| ---------- | ------------------------------------------------------------------ |
| Vision     | `prd`, `idea`, `mrd`, `brd`, `urd`, `brs`, `strs`, `syrs`, `srs`, `plan` |
| Knowledge  | `adr`, `rfc`, `rule`, `guide`, `doc`, `spec`                       |
| Experience | `task-type`, `cpat`                                                |

Full descriptions and required sections for each type: [Document types](/concepts/document-types/).
