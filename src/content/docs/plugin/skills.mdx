A **skill** is a unit of expertise the agent loads on demand. Each one carries focused knowledge — when to use a document type, required sections, typical relations, common mistakes — so you describe intent instead of schema. You invoke a skill via a `/archcore:*` slash command, or by simply describing what you want.

The plugin ships **18 skills**: 11 intent commands, 6 tracks, and 1 utility.

There is no per-document-type skill. The 18 [document types](/concepts/document-types/) (`adr`, `prd`, `rule`, `guide`, `spec`, …) are produced by intent commands and tracks, or — when you need a niche type without a routing skill — by calling `mcp__archcore__create_document` from the agent.

## All 18 skills at a glance

| Skill                            | Family   | What it does                                                            |
| -------------------------------- | -------- | ----------------------------------------------------------------------- |
| `/archcore:context`              | intent   | Load rules, ADRs, specs, and patterns that apply to a code area.         |
| `/archcore:bootstrap`            | intent   | First-time onboarding — scale-aware seeding for an empty `.archcore/`. |
| `/archcore:capture`              | intent   | Document a module / component / API — routes to adr, spec, doc, or guide. |
| `/archcore:plan`                 | intent   | Plan a feature — routes to `product-track` or a single plan.            |
| `/archcore:decide`               | intent   | Record a finalized decision — creates ADR, offers rule + guide follow-up. |
| `/archcore:standard`             | intent   | Establish a team standard — drives `standard-track`.                    |
| `/archcore:review`               | intent   | Documentation health audit — gaps, staleness, orphans, missing relations. |
| `/archcore:status`               | intent   | Counts dashboard — by category, type, status, relation coverage.        |
| `/archcore:actualize`            | intent   | Detect stale docs via code drift, cascade analysis, temporal signals.   |
| `/archcore:graph`                | intent   | Render the document relation graph as a Mermaid flowchart.              |
| `/archcore:help`                 | intent   | List every skill grouped by family.                                     |
| `/archcore:product-track`        | track    | idea → prd → plan.                                                      |
| `/archcore:sources-track`        | track    | mrd → brd → urd.                                                        |
| `/archcore:iso-track`            | track    | brs → strs → syrs → srs (ISO 29148).                                    |
| `/archcore:architecture-track`   | track    | adr → spec → plan.                                                      |
| `/archcore:standard-track`       | track    | adr → (optional cpat) → rule → guide.                                   |
| `/archcore:feature-track`        | track    | prd → spec → plan → task-type.                                          |
| `/archcore:verify`               | utility  | Plugin integrity check — configs, scripts, hooks, test suite.           |

## How skills activate

1. You type a slash command (`/archcore:context`) or describe an intent ("what rules apply to `src/auth/`?").
2. The host matches the request to a skill definition in the plugin package.
3. The skill is loaded into the conversation — its description, guardrails, and relation rules become part of the agent's working context.
4. The agent executes the skill, calling MCP tools as needed.

Every skill ultimately resolves to **MCP tool calls**. The plugin never writes to `.archcore/` directly.

## When to use which family

- **Start with an intent command** when you're not sure which document type fits. `/archcore:capture` and `/archcore:decide` will pick for you.
- **Use a track** when one document alone won't do the job. Need a decision _and_ a rule _and_ a how-to guide? Use `/archcore:standard-track`.
- **Run the utility** (`/archcore:verify`) when you suspect a plugin install issue — it checks configs, scripts, skills, hooks, and runs the test suite.

## Discovering skills at runtime

Inside any plugin-enabled session:

```
/archcore:help
```

Returns the same list grouped by family, with one-line descriptions. Useful when you forget the exact command name.
